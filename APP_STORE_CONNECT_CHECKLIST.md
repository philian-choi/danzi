# App Store Connect 제출 체크리스트

이 문서는 단지노트 앱을 App Store에 제출할 때 필요한 모든 항목을 체크하는 가이드입니다.

## 필수 정보 입력

### 기본 앱 정보
- [ ] **앱 이름**: 단지노트
- [ ] **Bundle ID**: `com.danzi.danzi` (확인 필요)
- [ ] **Primary Language**: 한국어
- [ ] **카테고리**: 생산성 (Productivity) 또는 부동산 관련 카테고리
- [ ] **서브카테고리**: (선택사항)

### 앱 설명
- [ ] **앱 설명** (최대 4000자):
  ```
  부동산 임장 기록을 체계적으로 관리하는 앱입니다.
  
  주요 기능:
  - 임장 기록 작성 및 관리
  - 사진 다중 선택 및 첨부
  - 검색 및 정렬 기능
  - 데이터 백업 및 복원
  
  모든 데이터는 기기에만 저장되며 완전히 오프라인으로 동작합니다.
  ```

- [ ] **키워드** (최대 100자, 쉼표 구분):
  ```
  부동산,임장,아파트,기록,메모,생산성
  ```

- [ ] **프로모션 텍스트** (선택사항, 최대 170자)

### URL 정보
- [ ] **지원 URL**: 웹사이트 또는 지원 페이지 URL
  - 예: `https://[yourdomain].com/support` 또는 GitHub 이슈 페이지
- [ ] **프라이버시 정책 URL** (필수): 
  - 예: `https://[yourdomain].com/privacy-policy`
  - PRIVACY_POLICY_TEMPLATE.md 참고하여 작성 필요
- [ ] **마케팅 URL** (선택사항): 앱 소개 페이지

## Export Compliance (암호화 선언)

App Store Connect 제출 시 암호화 사용 여부 질문:

### 질문: Does your app use encryption?
- [ ] **답변**: No (아니요)
  - 이유: Flutter 앱은 기본적으로 암호화를 사용하지 않으며, 
    앱 내 데이터는 평문으로 저장됩니다.

### 질문: Is your app exempt from encryption requirements?
- [ ] **답변**: Yes (예)
  - 이유: 일반적인 데이터 암호화 기능을 사용하지 않으므로 
    ITAR(International Traffic in Arms Regulations) 면제 대상입니다.

**참고**: 앱이 HTTPS를 통해 네트워크 통신을 하지 않으므로 
(오프라인 앱) 암호화 선언이 필요 없습니다.

## 빌드 업로드

### Archive 생성 전 확인
- [ ] Xcode에서 Release 빌드 설정 확인
- [ ] Code Signing 설정 확인 (Team, Provisioning Profile)
- [ ] Bundle Identifier 확인: `com.danzi.danzi`
- [ ] 버전 번호 확인: pubspec.yaml과 일치하는지

### Archive 및 업로드
- [ ] Xcode에서 Product > Archive 실행
- [ ] Organizer에서 "Validate App" 클릭하여 검증
- [ ] 검증 통과 후 "Distribute App" 클릭
- [ ] "App Store Connect" 선택
- [ ] 업로드 완료까지 대기

### App Store Connect에서 빌드 선택
- [ ] App Store Connect 접속
- [ ] 앱 > 버전 정보로 이동
- [ ] "빌드" 섹션에서 업로드된 빌드 선택
- [ ] "저장" 클릭
